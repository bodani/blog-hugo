##### 异地多机房策略

A. 当异地节点为一个节点。

- 备用机房节点在failover时不能选做主
- 备用机房节点主从复制采用异步方式

实现方法： 在yml 中的tag配置如下
```
tags: 
  nofailover: true # failover 时不能选为主节点
  nosync: true # 异步
```

```
patronictl -c /etc/patroni.yml list

| Member      | Host       | Role    | State   | TL | Lag in MB | Tags             |

| postgresql0 | 10.10.1.11 | Replica | running | 20 |         0 | nofailover: true |
```
